Common CSS Position Mistakes

Now let’s quickly look at some common mistakes beginners make with CSS position. Hover over the cards to see a small animation.

Mistake number 1

Using absolute without a relative parent

When you use position: absolute; without setting the parent to position: relative;, the element may move to an unexpected place.

Check whether the parent element has position: relative;.
Mistake number 2

Using sticky without top

When you use position: sticky; without a value like top: 0;, sticky usually does not work the way you expect.

Always add a sticky offset like top: 0; or top: 20px;.
Mistake number 3

Using top, left, right, or bottom on static

Static is the default position. So properties like top, left, right, and bottom do not work on a static element.

If you want those properties to work, change the element to relative, absolute, fixed, or sticky.
Final tip: If your positioning looks broken, these are the first three things you should check.
